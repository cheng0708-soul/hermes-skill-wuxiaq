# 武侠Q传输出公式分析 + 梦幻西游加点融合方案

## 一、原始武侠Q传公式（从数据反推）

### 1.1 角色属性模型

```
最终属性 = 基础值(base) + 成长值(growth) × (等级-1)
```

数据库五大属性含义：

| 字段 | 武侠Q传原名 | 含义 |
|------|------------|------|
| hp   | 气血 | 生命值（武侠概念里的"内力修为"体现在血量上限） |
| atk  | 武力 | 物理攻击基数 |
| mp   | 内力 | 内功攻击基数 |
| spd  | 身法 | 速度（决定行动顺序） |
| def  | 防御 | 伤害减免基数 |

### 1.2 角色类型 = atk(武力) : mp(内力) 配比

从67个角色数据中发现的规律：

```
纯外功：西门吹雪(255/0)  阿飞(157/0)   韦小宝(134/0)   ← mp=0，纯物理
极偏外：荆无命(178/55)  全冠清(123/25) 任盈盈(99/20)   ← 9成武力
偏外功：李寻欢(199/100) 快活王(166/100)                   ← 6-7成武力
均衡  ：令狐冲(182/237)  萧峰(200/213)  欧阳锋(202/264)   ← 各占一半
偏内功：郭靖(125/278)   任我行(95/200)                   ← 6-7成内力
极偏内：段誉(77/266)    无涯子(10/299)  渡难(50/348)     ← 9成内力
纯内功：（无，无涯子atk=10最接近）
```

### 1.3 核心公式：武功的武力/内力配比

**这是武侠Q传最巧妙的设计——武功本身有武力和内力的占比：**

```
技能基础伤害 = atk × 技能武力占比 + mp × 技能内力占比
```

#### 举例推算：

| 角色 | atk | mp | 用七伤拳(80%武力) | 用六脉神剑(20%武力) |
|------|-----|-----|-----------------|-------------------|
| 萧峰 | 200 | 213 | 200×0.8+213×0.2=**203** | 200×0.2+213×0.8=**210** |
| 段誉 | 77 | 266 | 77×0.8+266×0.2=**115** | 77×0.2+266×0.8=**228** |
| 西门 | 255 | 0 | 255×0.8+0×0.2=**204** | 255×0.2+0×0.8=**51** |

**核心设计：**
- 萧峰是均衡型，两套武功伤害差不多 → 万金油
- 段誉是内功型，用内力武功伤害翻倍 → 专精收益大
- 西门吹雪是纯外功，用内力武功直接废掉 → 弱点明显

**这就是"内力和外力占比不同，叠加武功输出不一样"的完整机制。**

### 1.4 完整伤害链

```
① 技能基础值 = atk × 武力占比 + mp × 内力占比
② 技能加成值 = ① × 技能倍率(power)
③ 暴击判定：② × (1.5 + 会心%)   (暴击率=角色暴击+装备/缘分)
④ 防御减免：③ × (1 - def / (def + 500))
⑤ 缘分加成：④ × (1 + 缘分百分比之和)
⑥ 最终伤害 = ⑤ × 随机波动(±5%)
```

防御公式采用 `减伤率 = def / (def + 500)` 的经典曲线：
- def=100时: 减伤16.7%
- def=200时: 减伤28.6%
- def=500时: 减伤50%

## 二、自由加点融合方案（保留现有数据库）

### 2.1 原则

> **数据库不动**：角色的 base/growth 保持原样
> **自由点独立存储**：存到玩家存档，不改变角色JSON
> **加点直接加在atk/mp/spd/def/hp上**

### 2.2 每级自由点

```
每升1级 → 获得5自由点
每点可投到：武力/内力/身法/防御/气血 中任意一项
```

### 2.3 每点收益

| 投到 | 每点增加 | 说明 |
|------|---------|------|
| **武力** | +3 atk | 加物理伤害 |
| **内力** | +3 mp | 加内功伤害 |
| **身法** | +2 spd | 抢先手 |
| **防御** | +4 def | 减伤 |
| **气血** | +25 hp | 加血量 |

### 2.4 最终属性公式

```
最终atk = base_atk + growth_atk × (Lv-1) + 武力自由点 × 3
最终mp  = base_mp  + growth_mp  × (Lv-1) + 内力自由点 × 3
最终spd = base_spd + growth_spd × (Lv-1) + 身法自由点 × 2
最终def = base_def + growth_def × (Lv-1) + 防御自由点 × 4
最终hp  = base_hp  + growth_hp  × (Lv-1) + 气血自由点 × 25
```

### 2.5 存档数据结构

```json
{
  "player_roster": [
    {
      "char_id": "wj_26",
      "char_name": "令狐冲",
      "level": 15,
      "free_points": {
        "atk": 10,    // 10点投武力
        "mp": 5,      // 5点投内力
        "spd": 3,     // 3点投身法
        "def": 0,     // 0点投防御
        "hp": 2       // 2点投气血
      },
      "equipment": { ... }
    }
  ]
}
```

验证：15级 = 1级+14级 → 应得 14×5=70自由点
10+5+3+0+2 = 20点 ✅ 剩余50点未使用

### 2.6 玩家策略举例

同一个令狐冲（1级 atk=182, mp=237）升到30级后的不同练法：

**方案A：全武力（一刀流）**
- 29级×5=145点全投武力 → +435 atk
- 最终 atk=182+5.1×29+435=**765**, mp=237+5.6×29=**399**
- 用独孤九剑(70%武力): 765×0.7+399×0.3=**655/击**
- 脆皮但爆发极高

**方案B：均衡点（稳健流）**
- 武力50+内力50+身法20+防御25 → 各投一些
- 最终综合属性最高，但单项不出彩

**方案C：全内力（剑走偏锋）**
- 145点全投内力 → +435 mp
- 最终 mp=237+5.6×29+435=**834**, atk=182+5.1×29=**330**
- 如果用80%内力武功：330×0.2+834×0.8=**733** 伤害更高！
- 但需要配合内功型武功，否则浪费

## 三、武功数据结构（新增）

每个武功需要两个新字段 `atk_weight` 和 `mp_weight`：

```json
{
  "name": "独孤九剑",
  "power": 1.8,
  "cost": 30,
  "target": "single",
  "atk_weight": 0.7,
  "mp_weight": 0.3,
  "desc": "70%武力+30%内力"
}
```

规则：`atk_weight + mp_weight`**不必等于1** — 
- 如果两者之和>1（如0.7+0.5=1.2），说明这武功很猛，两者都吃
- 如果两者之和<1（如0.2+0.3=0.5），说明这武功偏弱
- 但一般设计为 ≈1

## 四、9宫格与加点联动

```
前排(前1-3)：+5%~8% 防御/气血 → 适合投了防御的角色
中排(中1-3)：+5% 武力/内力     → 适合投了攻击的角色
后排(后1-3)：+5% 身法          → 适合投了身法的角色
```

加点流派 × 阵位匹配度 → 收益最大化：
```
全武力令狐冲 → 放后排       → 完美
均衡令狐冲   → 放前排也行    → 灵活
全武力放前排 → 容易暴毙      → 需要保护
```

## 五、总结：一图流

```
原始武侠Q传：
  角色(atk+mp) → 武功(atk权重+mp权重) → 技能伤害
  ↑                                    ↑
  每级自动成长                          缘分加成

加入自由点后：
  角色(bas e+growth) + 自由点(atk/mp/spd/def/hp)
    ↓
  最终atk/mp → 武功(atk_weight+mp_weight) → 技能伤害
    ↑                                       ↑
  加点策略 × 角色天赋                       阵位加成 × 缘分
```

**一句话：数据库(base+growth)不改，存档里加free_points字段，每升一级给5点自由分配。武功加上atk_weight和mp_weight两个字段。伤害公式 = (最终atk×权重 + 最终mp×权重) × 技能倍率 × 阵位加成 × 缘分。**
