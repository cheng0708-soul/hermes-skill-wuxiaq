# 角色生产流水线（SPUM 代码化产线）

> **这是"加武将 / 改外观"的唯一操作手册。** 照本文做即可，**不需要重新研究 SPUM、不需要开 SPUM 编辑器 GUI、不需要美术手工捏人**。
> 最后更新：2026-06-06（首版，随校对回路持续补注部件备注）。

---

## 0. 一句话原理

武将外观 = 一组「部件精灵 + 颜色」。我们用 Editor 脚本 `SPUMCharacterBuilder` **克隆一个现成 SPUM 样板 prefab**，按武将 JSON 里的 `art.spum_parts` 配方**换掉头发/脸/胡须/衣服/裤子/披风/武器的精灵和颜色**，存成 `Resources/Units/<id>.prefab`。动画/骨骼/spumPackages 全部继承样板、**绝不改动**。

> ⚠️ **重要认知**：早期文档曾说"造形必须人工 GUI、文字模型做不了"——**那是错的，已作废**。SPUM 角色外观是可序列化数据，能纯代码生成。

---

## 1. 三步傻瓜流程

### 第一步：写配方（改 1 个 JSON）
编辑 `Assets/Game/Resources/Characters/<id>.json`，在 `art` 块里加 `spum_parts`（和可选 `spum_template`）。例（萧峰 wj_03）：
```json
"art": {
  "spum": "契丹豪侠：壮硕、束发、短须、灰褐劲装+披风、徒手……",   // 中文设计稿，给人看
  "color": [0.82, 0.52, 0.28],
  "scale": 1.08,
  "spum_template": "hero_sword",      // 用哪个样板，省略=hero_sword（玩家朝向、右手持械）
  "spum_parts": {                     // ← 机器可读配方
    "Body":     { "name": "Human_3",   "color": "D2A679" },
    "Eye":      { "name": "Eye0",      "color": "3A2A1A" },
    "Hair":     { "name": "Hair_5",    "color": "2A211A" },
    "FaceHair": { "name": "FaceHair_1","color": "2A211A" },
    "Cloth":    { "name": "Cloth_4",   "color": "8C6B47" },
    "Pant":     { "name": "Foot_1",    "color": "5A4632" },
    "Back":     { "name": "Back_1",    "color": "6E5238" }
    // 徒手武将不写 Weapons；持械写 "Weapons": { "name":"Sword_2", "color":"DCE0EA" }
  }
}
```
规则：
- `name` 必须是部件库里存在的名字（见 §4 速查表）。
- `color` 是 6 位 hex（不带 #）。给 **Body=肤色、Hair=发色、Cloth=衣服色**——**不要**都填武将主题色，否则脸/手也被染。
- **省略某部件**：Body/Eye/Hair/FaceHair/Cloth/Pant/Armor/Back 省略 = 保持样板原样；**Weapons/Helmet/Armor/Back 省略 = 清空**（不显示）。徒手武将就别写 Weapons。

### 第二步：跑产线（1 条命令）
确认 Unity Editor **没开着**（batchmode 会锁工程），然后：
```bash
"/Applications/Unity/Hub/Editor/6000.4.10f1/Unity.app/Contents/MacOS/Unity" \
  -batchmode -quit -nographics \
  -projectPath "/Volumes/D2T/武侠小游戏/Unity/武侠q传unity" \
  -executeMethod WuxiaQ.EditorTools.SPUMCharacterBuilder.Run \
  -logFile /tmp/spum_build.log
grep -E "SPUMBuilder|缺图|error CS" /tmp/spum_build.log
```
- 脚本扫所有 `Characters/*.json`，**凡带 `spum_parts` 的都(重新)生成** `Units/<id>.prefab`（幂等，可反复跑）。
- 也可在 Editor 里点菜单 **「武侠Q/生成武将 SPUM 角色」**。
- 体检：`-executeMethod WuxiaQ.EditorTools.SPUMCharacterBuilder.Verify`（菜单「武侠Q/校验武将 SPUM 角色」）确认每个 prefab 存在、有 SPUM_Prefabs、身体精灵非空。

### 第三步：Play 视觉校对（人来看）
进游戏看角色。**这一步必须人来做**——脚本能装配但看不到像素长相。看哪不对 → 回第一步改 `spum_parts` 的 name/color → 重跑。常见反馈→改法见 §3。

## 2. 配方字段与流派对照

| PartType | 含义 | 省略时 | 备注 |
|---|---|---|---|
| Body | 身体(含头/四肢，6 子件) | 保持样板 | color=肤色 |
| Eye | 眼睛 | 保持样板 | |
| Hair | 头发 | 保持(样板可能无发) | color=发色 |
| FaceHair | 胡须 | 不显示 | 络腮/短须用 |
| Cloth | 衣服(Body/Left/Right 3 子件) | 保持样板 | color=衣服色 |
| Pant | 裤/腿(Left/Right) | 保持样板 | |
| Armor | 盔甲 | **清空** | 与 Cloth 二选一，武侠多用 Cloth |
| Helmet | 头盔 | **清空** | 武侠多不戴，留空露发 |
| Back | 披风/背挂 | **清空** | 大氅、箭袋等 |
| Weapons | 武器 | **清空** | 见下流派对照 |

**流派→武器对照**（按 combat.archetype / 兵器）：
- 近战拳掌（萧峰）→ **徒手**，不写 Weapons。
- 近战剑（西门吹雪/令狐冲）→ `Sword_1..6`。
- 法术内功（段誉/一灯）→ **徒手指剑**（不写）或法杖 `Ward_1`。
- 弓远程/暗器（楚留香）→ `Bow_1`（暗器无专属件，可徒手+轻装表现）。
- 枪 `Spear_1`/`Soon_Spear`、斧 `Axe_1`、盾 `Shield_1`、锤 `F_SR_Hammer`。

> 武器默认贴在样板的持械手（`hero_sword` 样板=右手 Dir=Right；脚本自动沿用样板已填武器的朝向）。

## 3. 视觉校对回路（用户反馈 → 改哪）

| 你看到/想要 | 改 `spum_parts` 的 |
|---|---|
| 发型不对（要高髻/散发/光头…） | `Hair.name` 换号；光头=删掉 Hair |
| 头发/胡须颜色不对 | `Hair.color` / `FaceHair.color` |
| 脸/手肤色发绿发白 | `Body.color`（别填主题色，填肤色） |
| 衣服款式/颜色不对 | `Cloth.name` / `Cloth.color` |
| 该有披风/不该有披风 | 加/删 `Back` |
| 武器换种/去掉 | `Weapons.name` / 删掉 Weapons |
| 体型太大太小 | 这是 `art.scale`（不在 spum_parts，BattleUnit 用它缩放） |

改完重跑第二步即可。

## 5. 样板（spum_template）

`Resources/Units/` 下现成可作样板的 prefab：
- `hero_sword` / `hero_spear` / `hero_fist`：玩家朝向，武器在右手。
- `enemy_bandit` / `enemy_archer` / `enemy_boss`：敌人朝向，武器在左手。

做武将统一用 `hero_sword` 即可。

---

## 6. 原理与排错

- **匹配机制**：脚本遍历样板 `SPUM_MatchingList.matchingTables`，按 PartType/Structure 把精灵贴到对应渲染器。
- **贴错图/某件没换**：找不到同名精灵会回退第一张。查 `Index.json` 该 Name 的 SubType。
- **外饰残留**：脚本已对 Weapons/Helmet/Armor/Back 整体清空。
- **动画没了/报错**：说明误改了 Animator/spumPackages——脚本只动 SpriteRenderer。
