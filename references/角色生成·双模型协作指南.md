## 一、双模型协作定位与分工

```
沙盒大模型（角色生成器）
  │
  ├─ 产出 wj_xxx.json         → 配方/数值/技能/流派
  ├─ 产出 wj_xxx.prefab       → SPUM 捏好的像素小人
  └─ 在沙盒中调试飞弹/VFX/飘字
       │
       ▼
主项目大模型（武侠Q传主AI）
  │
  ├─ 读取 JSON → 绑定卡池/属性/战斗AI
  ├─ 加载 Prefab → 调用 SPUMVisualAdapter 驱动动画
  └─ 套用参数化特效 → 渲染飞弹/粒子/震屏
```

### 各自职责

| 谁做 | 负责什么 | 不做什么 |
|------|---------|---------|
| **沙盒大模型** | SPUM捏脸、属性平衡、功法包装、飞弹颜色微调、在沙盒中验收效果 | 不碰卡池逻辑、战斗计算、AI行为树 |
| **主项目大模型** | 游戏大厅、卡池、战斗逻辑、伤害公式、AI策略 | 不碰像素人怎么捏、发型颜色、特效大小是否穿帮 |

## 二、导出与交付资产规范

沙盒大模型设计好一个角色后，产出两项核心文件：

### 资产一：角色配方 JSON

```
路径：Assets/Game/Resources/Characters/wj_xxx.json
```

### 资产二：SPUM Prefab

```
路径：Assets/Game/Resources/Units/wj_xxx.prefab
```

包含：SPUM 换装蒙皮骨骼（Hair/Cloth/Eyes/Weapon）+ 标准状态机 + OverrideController。
主项目 `Instantiate` 即可渲染小人。

## 三、主项目导入步骤（3步）

### 步骤A：解析数值 + 加载预制体

```csharp
TextAsset jsonAsset = Resources.Load<TextAsset>("Characters/" + characterId);
CharData data = JsonUtility.FromJson<CharData>(jsonAsset.text);

GameObject prefab = Resources.Load<GameObject>("Units/" + characterId);
GameObject go = Instantiate(prefab, parentAnchor);
```

### 步骤B：播放动作

```csharp
var visual = go.GetComponent<SPUMVisualAdapter>();
if (visual == null) visual = go.AddComponent<SPUMVisualAdapter>();

visual.PlayAnimation("Attack");   // 普攻
visual.PlayAnimation("Skill");    // 绝招
visual.PlayAnimation("Hit");      // 受击
visual.PlayAnimation("Die");      // 死亡
```

### 步骤C：挂接参数化特效

```csharp
Color GetVFXColor(string archetype) {
    return archetype switch {
        "暗器毒杀" => new Color(0.92f, 0.72f, 0.12f), // 亮铜黄
        "内功奇门" => new Color(0.25f, 0.55f, 0.98f), // 太极蓝
        "刚猛外功" => new Color(0.95f, 0.65f, 0.1f),  // 罗汉金
        "回春辅助" => new Color(0.29f, 0.82f, 0.50f), // 翠绿
        _ => new Color(0.85f, 0.85f, 0.9f)
    };
}

Vector3 GetProjectileScale(string archetype) {
    return archetype switch {
        "暗器毒杀" => new Vector3(0.3f, 0.05f, 1f),    // 细长飞针
        "内功奇门" => new Vector3(0.15f, 0.15f, 1f),   // 饱满法球
        "刚猛外功" => new Vector3(0.2f, 0.2f, 1f),     // 标准掌印
        "回春辅助" => new Vector3(0.12f, 0.12f, 1f),   // 轻灵飞羽
        _ => new Vector3(0.25f, 0.1f, 1f)
    };
}
```

## 六、参数化特效组合矩阵

```
发射源 → 飞弹样式 → 尾迹颜色 → 击中爆裂 → 飘字 + 震屏
```

### 飞弹弹体样式（4种）

| 样式 | 描述 | 经典搭配 |
|------|------|---------|
| **精铁暗器** (Arrow/Bolt) | 扁平细长，极速 | 唐门：亮铜黄飞针+绿雾尾迹 |
| **太极法球** (Sphere) | 高速旋转八卦能量体 | 武当：太极蓝光球+冰蓝流光 |
| **罗汉掌印** (Palm) | 厚重纯光掌劲 | 少林：金色掌印+赤金尾迹 |
| **斩击剑气** (Slash) | 半月形半透明剑气切片 | 华山：白/灰色月牙+淡白残光 |

### 尾迹参数（Trail Renderer）

- 颜色：可设任意 RGB
- 宽度：`0.01f - 0.25f`
- 消逝时间：`0.1s - 0.4s`

### 击中爆裂粒子

- 数量：15-30个受重力像素块
- 配色：太极幽蓝 / 少林灿金 / 唐门毒绿 / 鲜血暗红 / 剑气雪白

### 伤害飘字

| 类型 | 颜色 | 效果 |
|------|------|------|
| 普通伤害 | 黄色 | 向上弧线飘移渐隐 |
| 暴击 | 亮红加粗 | 带回弹缩放 + "暴击!" 字样 |
| 治疗 | 绿色 | 带"+"号 |

### 相机震屏

| 触发 | 时间 | 幅度 |
|------|------|------|
| 普攻 | 0.05s | 0.01f |
| 技能/暴击 | 0.15s | 0.03f |
| 大招 | 0.25s | 0.05f |

## 八、角色生成器操作流程

```
① 查 references/角色生成·全员设计表.json 或 .md
    → 确定角色流派(class)、武器类型(weapon_type)、飞弹形状(projectile_shape)

② 查 04_技能图标/ → 选对应流派的技能图标
   暗器→Assassin / 弓→Archer / 外功→Barbarian / 内功→Mage

③ 查 05_装备与道具图标/ → 选对应的装备/秘籍/丹药图标

④ 填 JSON：class + weapon_type + projectile_shape + combat.archetype

⑤ 生成 spum_parts（部件号+色码）→ 跑CLI出prefab

⑥ 在沙盒验收飞弹颜色/粒子/飘字/震屏

⑦ 跑同步脚本 → 主项目AI读取JSON自动绑定战斗系统
```

**核心原则**：沙盒模型负责"好看"，主模型负责"好玩"。分界线就是 JSON + Prefab 两个文件。
