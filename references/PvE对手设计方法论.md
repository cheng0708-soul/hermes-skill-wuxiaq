# PvE对手设计方法论

> 适用于武侠Q传前10关的PvE敌人模板设计
> 公式基础：原版攻防公式 atk²/(atk+def×3) + 化解机制
> 验证脚本见 `scripts/enemies_stage_1_10.py`

---

## 一、核心原则

### 1.1 难度阶梯

```
Lv1-3 (教学层)  →  轻松碾压，建立信心
Lv4-5 (过渡层)  →  算算也能赢
Lv6-7 (挑战层)  →  不思考会吃力
Lv8-10 (困难层)  →  必须认真排阵
```

每3关一个难度台阶，中间穿插「教学关」（新机制引入）。

### 1.2 敌人类型多元化

每个关卡应覆盖2-3个战斗维度：

| 维度 | 对应敌人 | 教学目的 |
|------|---------|---------|
| 速度 | 流寇spd=130+ | 先手威胁，需要高速角色应对 |
| 防御 | 武僧def=110+ | 普攻刮痧，需要内功/技能穿透 |
| 法术 | 魔教mp=180+ | 化解机制教学，低mp角色被克制 |
| 人数 | 4人队 | 集火优先级决策 |
| Boss | 高atk高HP | 综合考验 |

### 1.3 数值校验三铁律

```
① 任何单次伤害不超过玩家核心HP的12-15%
   → 验算：BOSS普攻74 / 萧峰640HP = 11.6% ✅
   
② 玩家击杀最硬敌人需3-6次普攻
   → 太短：敌人太脆没存在感
   → 太长：战斗拖沓
   
③ 随关卡递增的总HP增长速度应略快于玩家伤害增长速度
   → 玩家Lv1→Lv10：伤害约涨40%
   → 敌总HP：180→1713，约涨10倍（靠人数堆）
```

## 二、敌人模板设计标准

### 2.1 模板字段

每个敌人模板包含：

```python
{
    "lv": 基准等级（用于scaling基准值）,
    "hp": 生命值,
    "atk": 武力,
    "mp": 内力,
    "spd": 身法,
    "def": 防御,
    "skill": 技能（含wa/wm/ws权重）或None,
    "feature": "一句话说明用途"
}
```

### 2.2 等级缩放规则

敌人模板有自己的"基准等级"（lv字段），当放在不同关卡时：

```python
ratio = 1.0 + (target_lv - template_lv) * 0.05
# 每级+5%基础属性
```

### 2.3 PvE特有平衡考虑

**化解系统对PvE的影响**（这是个坑，第一次做时踩了）：

- 玩家角色多为甲级（高基础mp），导致敌人技能被大量化解
- 萧峰mp=213→245（Lv1→Lv10），几乎抵消所有低mp敌人的技能
- **解决方案**：敌人普攻atk顶上去（普攻不吃化解），技能走高mp权重
- **经验法则**：要把敌人技能伤害真正打出来，敌mp需达到玩家mp的60%以上
- 否则技能纯属装饰——又回到了只有普攻伤人的局面

**具体阈值**（以萧峰为参考）：

| 敌人mp | 对萧峰的化解效果 | 建议用途 |
|--------|----------------|---------|
| <60 | 技能≈0伤害 | 杂兵不要浪费技能，纯普攻 |
| 60-120 | 化解后剩少量 | 小精英用atk权重高的技能 |
| 120-200 | 化解后剩明显伤害 | 魔教教徒级别，真的疼 |
| 200+ | 少量化解 | BOSS级别，技能有威慑力 |

## 三、关卡设计清单

每次设计PvE关卡时确认：

- [ ] 该关卡的教学目的明确
- [ ] 敌人组合覆盖至少2个战斗维度
- [ ] 数值校验三铁律通过
- [ ] 标记是否为分水岭关卡（需要玩家加点/换阵）
- [ ] 文案与关卡名匹配（方便后续包装）
- [ ] 不做特殊机制（中毒/封印/治疗等属于P2）

## 四、已知坑

- **敌人atk不能太低**：atk²/(atk+def×3)公式中，atk过低时增长剧烈——atk从80到90能涨20%伤害，从200到210只涨5%。早期敌人atk的微小变化影响巨大
- **化解导致技能变0**：不是bug，但设计时要注意——如果敌人技能被完全化解，等于这个敌人只有一个普攻，很无聊。给高mp敌人配高mp权重技能
- **玩家加点不确定性**：无法预测玩家怎么加点。按「推荐加点」设计难度，让不加点的玩家吃些苦头，但不会卡关
