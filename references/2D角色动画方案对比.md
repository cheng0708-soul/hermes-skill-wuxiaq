# 武侠Q传 2D 角色动画方案对比

> 日期：2026-06-07
> 状态：参考文档（非定稿）

---

## 概述

本项目 2D 角色动画有三条技术路线：SPUM（当前在用）、UMotion Pro + Animancer Pro、AnyPortrait。三者能力上限和投入成本不同，可以组合使用。

---

## 方案 A：SPUM（当前方案 — 基础层）

### 是什么
Unity Asset Store 上的 2D 像素角色生成器。把角色分成头/身/手/脚/武器等部件，组合出不同外观。

### 能力上限
- ✅ 批量生成 64 个角色（已实现代码化产线）
- ✅ 提供基础动作：待机/行走/跑步/受击/死亡
- ✅ 风格统一（像素风 256×512）
- ❌ 没有武侠特色动作（没有出招/结印/大招）
- ❌ 骨骼简单，做不了复杂动画

### 适合
**角色身体生产**。SPUM 负责「64个人看起来不一样」，不负责「64个人动起来不一样」。

---

## 方案 B：UMotion Pro + Animancer Pro（动作增强层）

### 是什么
| 工具 | 作用 | 类比 |
|------|------|------|
| **UMotion Pro** | 在 Unity 编辑器里直接给任何骨骼角色调动作 | 像在 Unity 里内置了一个轻量 Maya |
| **Animancer Pro** | 用代码控制动画播放，不用拖 Animator 连线 | 一行代码替代一个状态机 |

### 能力上限
- ✅ 在 SPUM 已有的骨骼上**手搓任何动作**（出拳、挥剑、结印、浮空）
- ✅ 用 Animancer Pro 代码控制：`animancer.Play("六脉神剑")`
- ✅ 不需要外部软件，全部在 Unity 里完成
- ❌ 需要人手调，每个动作 ≈ 30分钟-2小时

### 适合
**为关键角色制作专属大招动作。** 不是给64人每人做10个动作，而是给核心6-8人各做1-2个招牌动作。

### 建议用法
```
核心角色（萧峰/段誉/令狐冲/西门/东方/虚竹）
  → UMotion Pro 搓专属大招动作（每人1-2个，≈ 6-8小时总工时）

其他角色
  → SPUM 通用攻击动作 + 技能特效覆盖
  → 玩家看到的是特效（剑气/掌波），小人动作通用
```

---

## 方案 C：AnyPortrait（2D 骨骼上限方案）

### 是什么
Unity 内最强的 2D 骨骼动画系统。支持网格变形（Mesh Deform），可以让角色衣服/头发自然飘动，脸部眨眼呼吸。

### 能力上限
- ✅ 可以做出媲美《阴阳师百闻牌》《崩坏学园2》的 2D 动画
- ✅ 支持网格变形，不露关节
- ✅ 自带 demo（你打开过的 SwordGirl 场景）
- ✅ 支持导入 PSD 分层文件

### 成本
```
1个角色 = 切片PSD + 绑骨骼 + 刷权重 + 搓动作 ≈ 1个工作日
64个角色 = 64天 ≈ 3个月全职
```

### 适合
- **少而精的角色**（比如只有6个核心角色）
- 不做成批量产线，而是手工打磨几个招牌角色
- 或者用作「大招插入动画」——段誉放大时切一个 AnyPortrait 独立动画播放，播完回到 SPUM 像素小人

```

AnyPortrait 是目前的上限方案，但不是必选项。
```

## 实战坑录（2026-06-07 补充）

### 坑1：SPUM + Animancer 集成的核心矛盾

**表象**：置空 SPUM 的 AnimatorController 后用 Animancer 直接播放 Clip → 人物卡死不动。

**根因**：SPUM 的 AnimatorController 不是纯粹的动画播放器，而是 **骨骼状态管理器**，承担以下职责：
- 武器挂点激活/隐藏（左右手切换、武器类型适配）
- 身体部件的遮罩/Layer 混合（换装、换色）
- 材质参数同步（透明度渐变动画、变色）
- 渲染组件的激活时序控制

强行置空 `runtimeAnimatorController` 会丢掉上述全部依赖链，骨骼寻址失效，人物自然静止。

**修复方案**：
1. 恢复调用 `_spumPrefab.OverrideControllerInit()`
2. 实现 `TryFindStateAndIndex(AnimationClip clip, out PlayerState state, out int index)`，智能检索 Clip 所属的 SPUM 状态（IDLE/ATTACK/DAMAGED 等）和数组索引
3. 调用原生 `_spumPrefab.PlayAnimation(state, index)` 驱动状态机，100% 保证动作播放正确
4. 对超出 SPUM 内置列表的自定义动画，**覆盖 OTHER 动画槽**作为兜底播放

**教训**：第三方骨骼插件的状态机不能当黑盒处理。必须先理解它通过 Animator 参数管理了哪些渲染/换装逻辑，再决定接入方式。

### 坑2：Unity 内置 UI 素材胜过手写颜色

**表象**：代码硬填 Dropdown 颜色值 → 显示为大白块，质感全无。

**修复方案**：
1. 使用 `DefaultControls.CreateDropdown` 默认样式，自动加载 Unity 内置 Dropdown 的背景和箭头贴图
2. 用 `AssetDatabase.GetBuiltinExtraResource<Sprite>("UI/Skin/Background.psd")` 载入内置圆角面板
3. 用 `AssetDatabase.GetBuiltinExtraResource<Sprite>("UI/Skin/UISprite.psd")` 载入内置立体按钮
4. Image.Type 设为 **Sliced**（九宫格拉伸），保证任意大小不糊边角
5. 配合半透明深色底色 `new Color(0.15f, 0.16f, 0.22f, 0.85f)`，呈现现代暗色 UI 质感

**教训**：Unity UI 的 Builtin Sprites 在绝大多数情况下够用且高质量。先看内置资源能不能满足需求，再考虑手写Style/找外置素材。代码硬改颜色通常是绕远路。
