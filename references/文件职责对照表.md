# 武侠Q传 · 文件职责对照表

> 每个系统对应一个或两个文件。改什么，去哪个文件改，一目了然。
> 最后更新：2026-06-08

---

## 一、角色/敌人数据

| 要改什么 | 去哪个文件 | 说明 |
|---------|-----------|------|
| 玩家武将属性 | `Resources/Characters/wj_XX.json` | stats/growth/skills/wugong/art/bonds |
| 玩家武将外观 | `Resources/Characters/wj_XX.json` → art.spum_parts | 捏脸配方，跑SPUM CLI生成prefab |
| 敌人属性 | `Resources/Characters/enemy_XXX.json` | 结构和武将JSON完全一样 |
| 新增一个武将 | 写一个 `wj_XX.json` + 跑 `tools/import_wugong.py` | 无须改C# |
| 新增一个敌人 | 写一个 `enemy_XXX.json` | 丢进 Characters/ 目录即可 |

## 二、武功系统

| 要改什么 | 去哪个文件 | 说明 |
|---------|-----------|------|
| 通用武功数据 | `resources-references/武功总表.json` | 109条，打宝掉落用，固定触发率 |
| 角色天赋武功 | `resources-references/天赋武功总表.json` | 85个角色专属，面板+spd补正→实战 |
| 角色武功槽关联 | `tools/import_wugong.py` | 把武功总表数据挂到角色JSON上，幂等可重跑 |

## 七、动画/特效管线

| 做啥 | 谁管 | 说明 |
|------|------|------|
| 角色捏脸 | SPUM CLI | 从JSON art.spum_parts 生成prefab |
| 技能动画 | SPUM AnimatorController | 通过 animTrigger 名驱动 |
| 技能粒子/飞弹 | VFX配置（武功特效表） | 跟武功走，不跟人走 |
| 演出文字 | JSON combat.cinematic | 当前仅备注，未接运行时 |

## 八、核心设计原则

```
改角色 → 写JSON           ← C#不动
改敌人 → 写JSON           ← C#不动
改关卡 → 写JSON           ← C#不动
改平衡 → 改combat_config  ← C#不动
改战斗逻辑 → 改C#         ← 频率最低
特效跟武功走，不跟人走     ← VFX ID存武功表
武功天赋版≠通用版，分离    ← 两个表不合并
```
