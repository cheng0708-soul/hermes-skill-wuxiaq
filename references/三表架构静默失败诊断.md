# 三表架构·静默失败诊断手册

## Bug 1：天赋武功不触发（JSON新旧共存）

### 症状
- 战斗中天赋武功（大招）全程不触发
- Console 无报错，`[CharacterImporter] 武功库未找到` 也没打

### 执行路径跟踪法

CharacterImporter.cs 的关键行：

```
L44:  def.ultimateSkill = 旧内联版(j.skills.ultimate)    → triggerChance=0.18
L124: baseDef = WugongLibrary.Get(talentName)             → triggerChance=0（库无此字段）
L131: talent = TalentWugongTable.GetFor(j.name)           → 如果 null，triggerRate 没覆盖
L147: def.ultimateSkill = baseDef                         → 双写清零！
```

### 根因
JSON 同时保留了 old `skills.ultimate`（内联，triggerChance=0.18）和新 `talent_wugong`（引用）。双写 L147 用库数据（triggerChance=0）覆盖了旧版 18%，导致永远不触发。

### 修复
1. 从 JSON 删除 `skills.ultimate` 字段
2. 从 JSON 删除 `wugong[]` 内联数组
3. 确保 `talent_wugong` 值精确匹配 wugong_lib.json 中的 name
4. 确保 Resources/Config/talent_wugong.json 有该角色条目且 trigger_rate > 0

### 预防
迁移角色时逐条走 `<三表角色JSON格式定稿.md>` 的检查清单。

## Bug 2：WugongLibrary 名字不匹配静默失败

### 症状
- Console 出现 `[CharacterImporter] 武功库未找到 '降龙十八掌'`
- 或 Console 无日志但天赋武功无 VFX/颜色（fallback 到 weapon 自动映射）

### 根因
JSON 中 `talent_wugong` 的字符串与 wugong_lib.json 中 entry 的 `name` 字段不完全一致（空格/标点/后缀差异）。

### 检查方法
```csharp
// 在 CharacterImporter 的 L124 前加打印
Debug.Log($"[CharacterImporter] 查武功库: name='{talentName}'");
var baseDef = WugongLibrary.Get(talentName);
Debug.Log($"[CharacterImporter] 查库结果: {(baseDef == null ? "null" : "OK")}");
```

## Bug 3：baseDef 跨角色突变污染

### 症状
- 角色 A 和角色 B 共用同一天赋武功（如萧峰/郭靖的降龙十八掌）
- 后加载的角色覆盖了前一个的 trigger_chance/attr 值

### 根因
L138-142 直接 mutate WugongLibrary 返回的缓存对象：
```csharp
baseDef.triggerChance = t.triggerRate; // 直接改了库里的缓存！
baseDef.atkWeight = t.atkWeight;
```

如果 WugongLibrary.Get() 每次都返回**同一个缓存对象**（没有 clone），第一个角色设置的值被第二个覆盖。

### 修复
```csharp
var baseDef = WugongLibrary.Get(talentName).Clone(); // 深拷贝
```

## Bug 4：TalentWugongTable 格式不匹配

### 症状
- L131 `talent.HasValue` = false
- triggerChance 保持为 0

### 根因
Resources/Config/talent_wugong.json 的解析格式与 GetFor() 查询方式不一致。

### 常见原因
1. JSON 格式不是代码期望的 keyed 结构（期望 `{ "characters": [...] }` 但给的是裸数组）
2. 角色名字匹配不上（JSON 是 `"萧峰"`，天赋表是 `"乔峰"`）
3. TalentWugongTable.cs 文件没创建或 GetFor 方法不存在

### 检查
```csharp
Debug.Log($"[CharacterImporter] {j.name}: 天赋表查询结果={(talent.HasValue ? $"触发率={t.triggerRate}" : "null")}");
```
