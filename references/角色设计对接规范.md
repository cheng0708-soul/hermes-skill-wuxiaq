## 0. 项目背景

《武侠Q传》复刻：2D像素横版，梦幻式指令回合制。

- 每回合指令(攻击/武功/防御) → 按速度(spd高者先)统一结算
- 天赋武功(大招)在攻击类指令时概率触发，怒气满100可手动强制触发
- 一方全灭分胜负，目标6~12回合

## 1. JSON Schema（完整字段）

id/name/title/tier/class/weapon/talent_wugong/role/faction/stats/growth/skills.normal/wugong[0~3]/equipment[null×4]/art/combat/lore

关键约束：
- skills 只含normal（不含ultimate/passive）
- talent_wugong 从69条名称表选取
- wugong[] 每条含name/power/powerStat/target/anim/cooldown/effectType/effectValue
- weapon 从枚举表选取，决定动画组
- art 不含spum_parts（Unity侧自动生成）
- combat.archetype 从流派表选取

## 2. 枚举表

weapon→动画组：拳掌→AttackMagicPush / 长剑短剑大刀长枪棍棒→Attack / 弓弩暗器→AttackBow / 拂尘法器扇笛→AttackMagic

archetype→飞弹：近战拳掌(金橙)/近战刀剑(银白)/斩击剑气(银白冷光)/刚猛外功(金橙重)/远程暗器(铜黄毒绿)/内功奇门(太极蓝)

effectType：damage/heal/shield/control/buff/debuff

spum_template：hero_sword(西门吹雪型)/hero_fist(萧峰型)/hero_magic(段誉型)/hero_bow(楚留香型)

## 3. 天赋武功名称表（69条）

可与 wugong_lib.json 对照，必须逐字匹配（含标点/空格/·），否则静默不触发。

## 4. 战斗规则

### 伤害公式
base = 倍率×(atk×w1 + mp×w2 + spd×w3)
伤害 = base² ÷ (base + def×3)
普攻永远物理 w1=1 w2=0 w3=0

### 天赋武功触发率  ← 🔴 重要修正
触发率 = trigger_rate + spd/1000 × 1.0  （身法系数=1.0，非0.1）

### 怒气机制
普攻+25 / 被击+10 / 大招后清零
满100可手动强制触发

### 武功冷却
wugong[] 有冷却回合，冷却中不可选

## 5. 数值参考区间

原版67将1级范围：hp 125-464 / atk 10-255 / mp 0-414 / spd 0-267 / def 0-200
成长范围：hp 2.6-11.4 / atk 0.4-10.8 / mp 0-12.3 / spd 0-8.1 / def 0-3.6
