# 两阶段指令制战斗系统（设计定稿）

> 日期：2026-06-07
> 状态：定稿 ✓（代码未实现）

## 为什么改

原版实时制下：
- 段誉普攻刮痧（atk=77），大招要看42%触发率的脸 -> 每回合没事干
- 萧峰、西门全程稳定平A -> 操作感太弱
- 玩家除了点大招按钮，没有任何策略选择

改成指令制后：
- 每回合选武功 -> 有策略感
- 段誉每回合能放一阳指（小伤害）或回血 -> 不再划水
- 六脉亮了就是人头 -> 保留武侠特色

## 核心设计约束

1. **不加大招的触发率**。段誉的大招应该是「一发定乾坤」而不是稳定输出。触发率保持原版水平（42%左右），一场能触发2-3次，每次约秒一人。
2. **不做蓝量池**。JSON里MP是法攻属性，不是蓝条。改 schema 代价太大，用冷却CD代替。
3. **武功槽数据驱动**。新加武功=改JSON，不改代码。沿用现有武功总表的数据。
4. **大招不打断当前回合指令**。怒气满了只标记就绪，下回合指令阶段让玩家主动选。

## 预留扩展方向（P2+）

- 封系（定身/昏睡/混乱）：使用武功总表已有武功名，effectType=Control
- 保护/护盾：effectType=Shield
- 被动天赋：补全弟子总表数据后接入 PassiveDef

## 不做的

- 蓝量池（不做，理由如上）
- 天赋树/宠物/修炼线（和64角色策略搭配无关）
- 冷却缩减/加速装备（P0不做，以后也不确定）
- 回合限时/自动跳过（P0不做，先保证体验）
